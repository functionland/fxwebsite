Although these payments are small and overall cheaper for users than traditional cloud, because there is no middle-man, all parties can get 13 times higher revenues compared with traditional cloud.

The project can be compared with both DePIN/AI and Storage protocols and has several key differences:

  • Plug and Play Hardware: We totally custom designed the hardware and did not use off-the-shelf ones. This not only reduces the cost of manufacturing, but also creates limitless opportunities for adding features and makes it plug and play due to having full control over both hardware and software. The hardware provides consumer-friendly dedicated nodes that can contribute to the resources and stability of the network.
  • Pricing: Due to the low-power node requirement for the network, the price of resources in the network is much less than any central or decentralized network. For example it costs half of Google or AWS to store data on Fula network and 1/10th of some web3 solutions.
  • Mobile-native SDKs: Mobile applications are the main way users interact with their data and Fula protocol supports both Android and iOS applications which is a huge improvement over other protocols which can only support web browsers.

The full application and data life cycle is as follows:

  1. Anyone can go to Google Play or App Store and download the applications built on top of the ecosystem (no need to be a node).
  2. When they login to the application a private key is generated and kept safely on the device. The key is generated from the login method, either MetaMask or traditional Google login approach.
  3. Any file intended to be uploaded to the network gets chunked into multiple pieces (through IPLD tree creation process), then each chunk is encrypted on the client device (through WNFS algorithm).
  4. Each encrypted chunk is then sent to a storage node in the network. Nodes are selected based on certain criteria to provide the user the best speed and uptime (through the Libp2p DHT).
  5. Then each piece is replicated 6 times on different nodes to ensure that data is reliably accessible (through IPFS-Cluster).
